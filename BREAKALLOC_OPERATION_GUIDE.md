```

  malloc()
  new()

  OnInitDialog()

```

```
        delete m_data;  // ← 看看这里有没有
}
```

## ⚠️ 常见问题

### Q: 按F5后没有中断？
**A:** 检查：
- 编译是否成功？
- 块号是否正确？
- 是否在Debug配置下运行？

### Q: 中断在malloc，看不到我的代码？
**A:** 这是正常的。点击调用栈中更高的行，找您的代码。

### Q: 修复后还是中断？
**A:** 修复可能不完整。查看：
- 是否所有的new都有delete？
- 是否是new[]需要delete[]？
- 是否还有其他释放路径遗漏？

### Q: 不知道怎么修复？
**A:** 参考 `START_HERE_MEMORY_LEAK.md` 的常见修复模式部分。
