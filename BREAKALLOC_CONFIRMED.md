## 💡 关键要点

### BreakAlloc的作用
```
_CrtSetBreakAlloc(块号);
    ↓
程序启动 → 分配内存 → 检查是否是目标块号 → 如果是，中断调试器
    ↓
您可以查看：
- 调用栈（谁分配的？）
- 变量值（为什么分配？）
- 源代码（在哪里？）
```

### 为什么要逐个追踪？
```
一次7个 → 中断7次 → 效率低 ❌
一次1个 → 集中处理 → 效率高 ✅
修复后再换 → 逐个验证 → 更可靠 ✅
```
